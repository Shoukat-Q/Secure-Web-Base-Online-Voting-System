1. Introduction.
The online voting system is a web base application to facilitate secure, efficient, and transparent voting through  the internet. Traditional voting methods often involve logistical challenges, long queues, and potential human errors.

2. Problem Statement.
  It suffer from several limitations:
  
  I. Time consuming manual processes.
  II. High operational costs.
  III. Risk of human errors.
  IV. Delayed result processing.

3. Objectives.
  The main objectives of the system are:

 I. To develop a secure authentication system for voters.
 II. To ensure one person can vote once.
 III. To design an admin panel for managing elections and candidates.
 IV. To ensure data integrity and system security.

4. Scope of Project.

  Voter Module:
  
  I. User registration and login.
  II. Viewing candidate list.
  III. Casting vote securely.
 IV. Vote confirmation

  Admin Module:

  I. Admin login.
 II. Candidate management(add/update/delete).
 III. Election management.
 IV. Voter management.
5. Proposed Methodology.

 Development Approach:

 I. Requirement gathering.
 II. System design(UML diagrams).
III. Implementation(Coding).
IV. Testing and validation.

 Technologies:

 I. Frontend: HTML, CSS, JavaScript, Bootstrap.
 II. Backend: Python(Django Framework)
 III. Database: MySQL
